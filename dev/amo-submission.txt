Copy/paste the relevant sections into the Firefox Add-ons submission form.


NAME
----
URL Lookalike Blocker


SUMMARY  (max 250 characters, no HTML)
-------
Protects against IDN homograph attacks by blocking or warning when a domain contains characters from scripts you have not permitted, or characters that visually resemble a different character.


DESCRIPTION  (HTML is supported)
-----------
<p>Protects against IDN homograph attacks — attempts by attackers to register domain names that look identical to legitimate sites by substituting visually similar characters from other Unicode scripts (for example, Cyrillic <b>а</b> in place of Latin <b>a</b>).</p>

<p>When you navigate to a URL, the extension decodes the hostname from punycode and checks every character. Three things can happen:</p>
<ul>
  <li><b>Blocked</b> — a character belongs to a script that is not in your permitted set.</li>
  <li><b>Warning</b> — all characters are permitted, but either (a) the label contains a known confusable character that mimics a Latin letter in a mixed-script context (highlighted red with "Looks like: …"), or (b) the label mixes characters from two or more scripts in a way that is not common for any single language (highlighted amber, with a hint suggesting which language to enable if the combination is legitimate).</li>
  <li><b>Allowed</b> — everything checks out.</li>
</ul>

<p><b>Blocked and warning pages</b> show the punycode and Unicode forms of the domain, and a table of every suspicious character with its Unicode codepoint, script, and what it visually resembles.</p>

<p><b>From a blocked page</b> you can allow the domain permanently, go back, or open settings to enable the required language. If you open settings from a blocked page, clicking Apply automatically retries the blocked URL.</p>

<p><b>From a warning page</b> you can allow the domain permanently, continue for this browser session only, go back, or open settings.</p>

<p><b>Options page</b> (opened from the toolbar icon) lets you enable additional permitted languages, manage your whitelist of trusted domains, and adjust interface options. Language and whitelist changes are held in memory until you click Apply — there is no auto-save for security-relevant settings. Interface options (Show shadows, Show private-browsing warning) take effect instantly.</p>

<p><b>Multiple blocked or warning tabs at once</b> are tracked with a numeric badge on the toolbar icon and coloured rounded squares in the Options page, so you can switch between them and resolve each one. Resolved or closed tabs clear from the badge automatically.</p>

<p><b>Help page</b> with screenshots and explanations of every feature is available from the toolbar icon's right-click menu.</p>

<p><b>Themes:</b> light, dark, or follow-system, with a toggle on every extension page.</p>

<p><b>Known limitations:</b></p>
<ul>
  <li>The extension detects single characters that look like a different character. It does not detect multi-character sequences that resemble one character (e.g. rn → m, vv → w, cl → d), as doing so without a list of known legitimate domains would cause too many false positives.</li>
  <li>It does not detect lookalike domains constructed entirely from one script (e.g. a Latin-only domain designed to visually mimic another Latin domain). Protection is specifically against cross-script substitution attacks (IDN homographs).</li>
  <li>It does not detect subdomain-style phishing where a legitimate-looking name appears as a subdomain of an attacker-controlled domain (e.g. apple.com.attacker.com). That domain is Latin-only and passes all script checks correctly.</li>
</ul>

<p>Firefox only — Chrome's Manifest V3 removes the blocking webRequest API required for this extension to work.</p>

PRIVACY POLICY
--------------
URL Lookalike Blocker collects no data. All processing happens locally in
your browser. No information about the URLs you visit, the domains the
extension blocks or warns about, or your extension settings is transmitted
to any server.

What is stored locally (via browser.storage.local):
  - The set of languages whose Unicode scripts you have permitted beyond
    your locale defaults.
  - The list of domains you have explicitly whitelisted.
  - Your interface preferences (theme choice; whether drop shadows are
    shown; whether the private-browsing warning is dismissed).

A small localStorage cache holds two of those preferences (your theme and
the show-shadows setting) so the next page-load can apply them
synchronously and avoid a brief flash of the previous theme. This cache
contains only those two values and never leaves your browser.

Everything listed above is stored on your device only and is removed if
you uninstall the extension. The extension does not modify page content,
inject scripts into pages, read browser history or cookies, or use any
third-party analytics, tracking, or telemetry.

Contact for privacy questions: open an issue at
https://github.com/feldegast/url-lookalike-blocker/issues or email
aussiefeld@gmail.com.

NOTES FOR REVIEWER  (the AMO reviewer notes field)
------------------
This extension uses webRequest with blocking, which requires the "webRequestBlocking"
permission. This is necessary because the extension must inspect and potentially
redirect the request before the page loads — there is no way to implement IDN
homograph detection without intercepting the request synchronously.

The "menus" permission adds two items to the extension icon's right-click menu:
"Open Options" and "Help". Both open extension-owned pages only.

The extension does not modify page content, inject scripts into pages, or
access page data. It only inspects the hostname of navigation requests.

No data is collected, logged, or transmitted. User settings (permitted
languages, whitelisted domains, interface preferences) are stored locally
via browser.storage.local. A small localStorage cache mirrors the theme
and shadow preferences so the next page-load can apply them synchronously
and avoid a flash of the previous theme; this cache holds only those two
preference values and never leaves the browser. The manifest's
data_collection_permissions is declared as "none".
