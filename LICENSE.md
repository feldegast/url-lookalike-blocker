## Why Dual License?

This dual-licensing approach provides flexibility:

- **GPL 3.0** offers strong copyleft protection, ensuring derivative works remain open source
- **MPL 2.0** provides a more permissive option that's compatible with proprietary software integration (such as browser implementations) while still requiring modifications to the licensed code to be shared

Both licenses ensure that:
- The source code remains accessible
- Modifications and improvements are shared back with the community
- The software cannot be made proprietary without sharing source code changes

## How to Choose

**Choose GPL 3.0 if:**
- You want to create derivative works that must also be fully open source
- You're building applications or services that will be distributed to users
- You want the strongest copyleft protection

**Choose MPL 2.0 if:**
- You want to integrate this code into a larger proprietary system (like a browser)
- You prefer file-level copyleft (only modified files need to be open sourced)
- You need more flexibility in combining with other licenses
