## Project Overview

**url-lookalike-blocker** is a Firefox browser extension (Manifest V3) that protects users from IDN homograph attacks by blocking navigation to domains containing characters outside the scripts associated with the user's locale. The extension is fully implemented and working.

**Firefox only.** Chrome's MV3 removes blocking `webRequest`, which is required for dynamic Unicode script detection.

| File | Role |
|---|---|
| `manifest.json` | MV3 manifest; declares `webRequest`, `webRequestBlocking`, `storage`, `tabs`, `<all_urls>` |
| `background.js` | Request interception; whitelist lookup; redirect to blocked page; tracks last blocked tab for retry |

## Core Detection Flow

1. `browser.webRequest.onBeforeRequest` fires with the request URL
2. Extract and punycode-decode the hostname
3. Check each character's Unicode script via `/\p{Script=...}/u`
4. Permitted scripts = locale scripts (`navigator.languages`) + user-enabled extras from storage
5. Always permit: **Common**, **Inherited**, **Latin**
6. If any character's script is not permitted → redirect to `blocked.html`
7. Whitelist check (decoded Unicode hostname) runs before the script check

## Key Design Decisions

- **Explicit Apply**: the options page holds changes in memory and only saves on Apply — no auto-save on checkbox change, no timers, no race conditions
- **Toolbar icon opens a full tab**: popups are destroyed on focus loss, which would discard unsaved option changes
- **Last blocked tab tracking**: `background.js` stores the tab ID of the most recently blocked navigation so the options page can navigate it back to the URL after applying new permissions
- **Whitelist storage**: decoded Unicode form (human-readable); matched against decoded hostname at request time
- **Script detection**: ES2018 Unicode property escapes (`/\p{Script=Cyrillic}/u`) with a fast manual range check for Cyrillic as a fallback
