URL Lookalike Blocker — Help

What does this extension do?

When you visit a website, your browser shows the domain name (like google.com). Attackers can register domains that look identical to real ones by substituting characters from other languages — for example, using a Cyrillic а instead of the Latin a in apple.com. These are called IDN homograph attacks.

This extension inspects every domain name your browser visits and blocks or warns you when it detects characters that are likely part of a spoofing attempt.

Blocked Block page

A block page appears when the domain contains characters from scripts (writing systems) that you have not enabled in settings. For example, if a domain contains Cyrillic characters and you have not enabled Russian, it is blocked.

What you can do

  • Allow this domain — adds the domain to your whitelist so it is never blocked again.
  • Open extension options — opens the Options page so you can enable the relevant language.
  • Try again — re-attempts the URL (useful after applying new settings).
  • Go back — returns to the previous page.

Warning Warning page

A warning page appears when the domain mixes scripts in a way that suggests a homograph attack — for example, a domain that looks Latin but contains one hidden Cyrillic character. Warning pages can appear even if you have that script enabled, because legitimate sites do not mix scripts within a single domain label.

The offending character is highlighted in red. If it only looks like a Latin character, that is shown too (e.g. "Looks like: a").

What you can do

  • Allow this domain — permanently whitelists the domain if you know it is safe.
  • Open extension options — opens the Options page.
  • Try again — re-attempts the URL after applying new settings.
  • Go back — returns to the previous page.

Options page

Permitted languages

Tick the languages whose scripts you want to allow in domain names. Your browser's locale is used as the starting point — so if your Firefox language is set to English, Latin is already permitted.

Enabling a language does two things: it allows that language's scripts in domain names, and it permits mixing those scripts within a single label (useful for languages like Serbian that legitimately combine Latin and Cyrillic).

Apply changes / Discard changes

Changes to language settings are held in memory until you click Apply changes. The orange "Unsaved changes" dot appears while there are pending edits. Discard changes reverts to the last saved state without closing the page.

Reset to locale defaults

Re-checks the languages that match your current Firefox locale setting, discarding any manual selections. Applied immediately.

Whitelisted domains

Domains you have explicitly allowed (via "Allow this domain" on a block or warning page) are listed here. You can remove them at any time — the domain will be blocked or warned again on the next visit.

Coloured squares

When multiple blocked or warning pages are open at once, each tab gets a unique coloured rounded square so you can tell them apart.

  • Clicking the square on a blocked or warning page opens the Options page.
  • From the Options page, clicking a square switches browser focus to that blocked tab.
  • When a blocked tab is closed or successfully loads its URL, its square disappears automatically.

When you click Apply changes in Options, the page closes and focus returns to the blocked tab that opened it.

Extension icon (toolbar)

Left-clicking the extension icon in the toolbar opens the Options page. Right-clicking gives a small menu with Open Options and Help.

If the Options page is already open, clicking the icon switches to it rather than opening a second one.

When one or more blocked or warning pages are open, the icon shows a red number badge indicating how many there are. The badge clears automatically once all blocked tabs have been resolved or closed.
